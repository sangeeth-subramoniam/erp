''' ようこそ！
これは、バックエンドでDjango Frameworkを使用し、フロントエンドとしてJavascriptを使用して構築されたモデルERP / CRMアプリケーションです...

このアプリケーションの目的は、組織内のさまざまなプロセスを自動化することです。シンプルなユーザーインターフェースで意図的に構築されているので、ユーザーフレンドリーです...
さまざまなプロセスをデジタル化および自動化することにより、多くの時間が節約され、作業の効率が向上します。
以下に、アプリケーションのさまざまなモジュールの実用的な説明を示します。


1サインアップ/サインイン：

Djangoの強力な認証システムを使用すると、サインアップとサインインは非常にシンプルで信頼性が高く、多くのセキュリティ機能を備えています。
世界の多くのトップ企業がDjangos認証システムを使用しており、非常に安全で信頼性が高くなっています。
世話をされる一般的なセキュリティ対策は、
1 XSS（クロスサイトスクリプティング）保護
2 CSRF（クロスサイトリクエストフォージェリ）保護
3SQLインジェクション保護
4クリックジャッキング保護
5安全なパスワードハッシュ

Django認証システムの詳細については、次のドキュメントを参照してください。
https://docs.djangoproject.com/en/stable/topics/security


2管理モジュール：

Djangoの管理モジュールは、アプリケーションでさまざまなアクセス権やその他のプロセスを制御および構成するためのシンプルなインターフェースを提供します

3通知：

このモジュールは、管理者がすべての従業員に情報を送信/ブロードキャストするために使用します
（ユーザーアクセス権を持つすべての人が通知を表示できます）

4チケット：

このモジュールは、従業員が管理者に提起して発行したい場合に使用されます。
従業員と管理者のみが自分の上げたチケットを表示できます
管理者は、チケットのステータスを反映するために、チケットに[PENDING / APPROVED]のマークを付けることができます

5出席：

このモジュールは、従業員の出席をマークするために使用されます...
アプリケーションでは、当日の出席をユーザーがマークすることができます。当日以外の日については、ユーザーはTICKETMODULEを介して管理者に連絡して出席をマークする必要があります。

6ダッシュボード：

ダッシュボードモジュールは、組織のメンバー内のさまざまなディスカッションで使用されます。これは、管理者だけでなくすべての従業員が参加できるグループチャットのように機能します

7メッセージ：

メッセージングモジュールは、従業員間のプライベートメッセージングシステムです。
メッセージは、従業員から従業員に、従業員から管理者に、またはその逆に送信できます。

8予約：

このモジュールは、組織のメンバー内で会議室、会議室、さらには周辺機器などのリソースを共有するために使用されます
このモジュールは、共有リソースの詳細と予約の詳細を表示するインターフェースを提供します。また、ユーザーが新しい予約をすることができるインターフェースがあり、新しい予約がすでに予約された時間に干渉する場合、新しい予約を行うことはできません
右上隅のユーザー名をクリックすると、名前、年齢、性別、部門情報、指定など、サインアッププロセス中に指定されたユーザーに関する基本情報が表示されます。

''' Welcome!


This is a model ERP/CRM application built using Django Framework in the backend and Javascript as the frontend...
The objective of the application is to automate various process in an organisation. Intentionally built with a simple user interface , so that is is User friendly...
By digitalising and automating various processes, lot of time is saved and the efficiency of the work is increased.
Below you will find the working explanation of various modules in the application,
1 SIGNUP/SIGNIN :

Using Django's powerful authentication system, Signing up and signing in is very simple and very reliable with many security features.
Many Top Companies in the world use Djangos Authentication system , making it very secure and reliable.
Common security measures taken care are,
1 XSS (cross-site scripting) protection
2 CSRF (cross site request forgery) protection
3 SQL injection protection
4 Clickjacking protection
5 Safe password hash

To learn more on Django authentcation system, Please refer the following documentation,
https://docs.djangoproject.com/en/stable/topics/security/

2 ADMIN MODULE:

Django's admin module gives a simple interface to contol and configure various access rights and other processes in the application
3 NOTIFICATION:

This module is used by the management to send / broadcast and information to all the employees
(Everybody with a user access can view the notifation)
4 TICKETS:

This module is used when a employee wants to raise and issue to the admin.
Only the employee and the admin can view his/her raised tickets
The admin can mark the ticket with [PENDING/APPROVED] to reflect the status of the ticket
5 ATTENDANCE:

This module is used for marking the attendance of employees...
In the application, the attendance of the current day can be marked by the users. For any day other than the current day, The User will have to contact admin through the TICKET MODULE to mark the attendance.
6 DASHBOARD:

The Dashboard module is used in various discussions within the members of the organisation. It acts like a group chat where every employee as well as the Admin can participate
7 MESSAGING:

The messaging module is a private messaging system between employee.
Messages can be sent from employees to employees , employees to Admin and vice versa
8 RESERVATION:

This module is used for sharing resources like Meeting rooms, conference rooms and even peripherals within the members of an organisation
The module gives an interface which displays the details of the shared resource and the reservation details. It also has an interface where an user can make a new reservation and if the new reservation interferes with an already reserved time , the new reservation cannot be made
Clicking on the Username in the right top corner gives the basic information about the user given during the Signup process like name, age, sex, Department information and designation.
